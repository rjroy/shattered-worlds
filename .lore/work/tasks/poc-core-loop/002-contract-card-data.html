What

Define the immutable types the whole architecture rests on — no behaviour yet, just the shapes everything else is checked against.

  • src/core/types.ts — concrete schemas:
type CardId = string            // "card-01" … "card-10"
interface Card { id: CardId; value: 1 | 2 | 3 }

type Action =
  | { type: 'PlayCard'; cardId: CardId }
  | { type: 'EndTurn' }

type GameEvent =
  | { type: 'CardPlayed'; cardId: CardId; value: number; runningTotal: number }
  | { type: 'TurnEnded'; total: number }
  | { type: 'CardsDiscarded'; cardIds: CardId[] }
  | { type: 'DeckShuffled' }
  | { type: 'CardsDrawn'; cardIds: CardId[] }

interface RngState { a:number; b:number; c:number; d:number }

interface GameState {
  drawPile: readonly Card[]   // top = index 0
  hand: readonly Card[]
  played: readonly Card[]     // in play order
  discard: readonly Card[]
  runningTotal: number
  history: readonly number[]
  rng: RngState
}
  • src/core/errors.ts — class IllegalActionError extends Error carrying readonly action: Action and readonly state: GameState (the single error contract).
  • src/core/cards.ts — buildLibrary(): Card[]: exactly five value-1, three value-2, two value-3 cards, ids card-01…card-10 in a fixed deterministic order (value sum 13).
  • src/core/contract.ts — type re-exports + the GameCore interface (pure declaration, no bodies). reduce/createGame implementations arrive in Step 4.

RngState is defined here, not in rng.ts, so GameState typechecks now; Step 3's rng.ts imports it. All array fields are readonly. Follow typescript-quality: discriminated unions over enums, no any, no non-null assertions.

Why

REQ-POC-2 (immutable state snapshots), REQ-POC-4 (exact ten-card composition, unique stable ids), REQ-POC-5 (four zones + running total + history + PRNG cursor modelled in state), and the REQ-POC-8 error contract all begin as type definitions here. Locking the shapes first lets every later step typecheck against a fixed contract. See plan Step 2.

Headless sim runner — the second driver

REQ-POC-14

What

Prove the "one core, two drivers" property: the same core, driven at full speed with no renderer, survives a long random-legal run while conservation holds.

  • src/sim/policy.ts — pure action chooser: given a state and an rng draw, if the hand is non-empty choose uniformly among playing each individual hand card or ending the turn; if the hand is empty, end the turn. Because a fresh 5 is dealt after every EndTurn, it never selects an illegal action.
  • src/sim/run.ts — drive createGame(seed) through the policy for a configurable count (default ≥1000), asserting the conservation invariant every step; exit non-zero on any throw or violation. Runnable via bun run sim.

The runner ignores the event log entirely — it consumes only state — demonstrating the renderer is optional to the core. The policy's own choice randomness uses the seeded core RNG helpers so a sim run is itself reproducible.

✓ Validation (bun test + bun run sim)

▸A test dispatches ≥1000 random-legal actions through the core alone; completes without throwing and preserves conservation throughout.
▸bun run sim exits 0 and prints a short summary (turns, actions, final history length).
▸The runner imports nothing from src/game/ or phaser (no renderer in the loop).

Why

REQ-POC-14 asks for a headless runner that drives the core without the renderer, proving the "one core, two drivers" property the architecture depends on for future balance data. The conservation assertion across a long run is also the strongest stress test of the Step 4 reducer. See plan Step 5.

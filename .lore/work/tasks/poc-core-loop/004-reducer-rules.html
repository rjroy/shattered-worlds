What

Implement the full turn loop as a pure reducer returning { state, events }, plus the stateful GameCore façade the renderer drives.

  • src/core/draw.ts — internal draw(state, n): pull n from the front of the draw pile; if it empties before n, shuffle the discard (seeded) into a new draw pile, emit DeckShuffled, continue. Returns new zones + CardsDrawn{cardIds} (+ optional DeckShuffled) and the advanced rng.
  • src/core/reduce.ts — reduce(state, action): { state, events }:
    • PlayCard — if cardId not in hand, throw new IllegalActionError(action, state); else move hand→played (append), add value to running total, emit CardPlayed{cardId,value,runningTotal}.
    • EndTurn — strict order: (1) append running total to history; (2) move hand + played into discard; (3) reset running total to 0; (4) draw 5 (reshuffle as needed). Emit TurnEnded{total}, CardsDiscarded{cardIds}, optional DeckShuffled, CardsDrawn{cardIds} in that order. Legal even when hand + played are empty (records 0).
  • src/core/game.ts — createGame(seed): GameCore: build library, shuffle (seeded), draw opening 5, total 0, empty history. The façade holds state and exposes dispatch(action) that calls reduce, stores the new snapshot, returns {state, events}.

A private conservation check (four zones = ten unique ids) is reused by tests and the sim runner. Events carry specific card ids so the renderer animates per card without diffing states. The design's config param is omitted for the POC.

✓ Validation (bun test — no Phaser/DOM/canvas)

▸New game: draw pile 5, discard 0, total 0, history empty, hand of 5.
▸PlayCard sums correctly; playing a card not in hand (unknown / already played / discarded) throws IllegalActionError carrying action + state.
▸EndTurn: history gains exactly one entry equal to the turn total, running total resets to 0, hand refills to 5, exactly one seeded reshuffle fires (observable via DeckShuffled).
▸EndTurn always legal: ending a turn with an empty hand + empty played pile records a total of 0 and redraws normally (REQ-POC-10).
▸Conservation invariant holds across a multi-turn sequence.
▸Replay equivalence: same seed + same action list ⇒ deep-equal final state and deep-equal event log; a second, different seed yields a different initial shuffle (≥2 seeds).
▸Synchrony: dispatch(...) returns a plain object, not a thenable (assert no .then).

Why

This step implements the loop's rules: REQ-POC-6 (new game), REQ-POC-7 (draw + reshuffle), REQ-POC-8 (play summation + throw-on-illegal), REQ-POC-9 (EndTurn strict order + events), REQ-POC-10 (no end condition). It honours REQ-POC-2 (synchronous, immutable dispatch) and REQ-POC-11 (id-carrying events), and its tests satisfy the headless-core half of REQ-POC-13. See plan Step 4.

P3 — Phaser preload + scene-init wiring (load JSON, surface failures)

Sequence 4 of 6  ·  Plan: card-data-externalization

Depends on: 003. Parallelizable with task 005. Trickiest wiring point — only the scene/boot/main files change; edits stay in distinct methods.

What

Wire the Phaser scene to load both JSON files in preload() and assemble the catalog in create() before any game state exists. worldId arrives as scene init data (it cannot be read from state in preload()).

  • Boot scene for init data: the current scene: [TableScene] auto-starts with no data path. Fix:
    • src/game/TableScene.ts — super({ key: 'Table' }) in the constructor.
    • src/game/BootScene.ts (new, ~10 lines) — super({ key: 'Boot' }); create() calls this.scene.start('Table', { worldId: 'zombie-big-box', seed: 42 }).
    • src/game/main.ts — scene: [BootScene, TableScene].
  • src/game/assetManifest.ts — add two ?url JSON imports/entries ('world-starter', 'world-zombie-big-box').
  • src/game/TableScene.ts:
    • init(data) — store this.worldId_ / this.seed_ (defaults 'zombie-big-box' / 42).
    • preload() — selectTheme(this.worldId_) for art keys; this.load.json('world-starter', ...) + this.load.json('world-' + this.worldId_, ...); register this.load.on('loaderror', ...) to set a failure flag.
    • create() — if the failure flag is set or this.cache.json.get(...) is undefined for either key, console.error and throw CatalogError before building state. Otherwise read both JSON, assembleCatalog([starter, world]), build the WorldData run-setup (worldId+deckComposition from world JSON, starterDeck from starter.json), then createGame(catalog, world, this.seed_); keep selectTheme(state.worldId) for the active theme.
  • Optional: extract a pure helper that maps a parsed world JSON → WorldData so the scene method stays a thin adapter and the mapping is unit-testable.

Validation — automated

  • Suite: bun test green (headless parts; assembly already covered by 002/003).
  • If the helper is extracted: a unit test maps a parsed world JSON to WorldData with correct worldId, starterDeck, and deckComposition.

Validation — manual (runtime, browser)

  • Boot + run: the app boots through Boot → Table and a full Zombie Big-Box run plays with cards minting correctly (REQ-CARD-DATA-2).
  • Load failure surfaced: temporarily rename/corrupt starter.json → a console.error appears and a CatalogError is thrown before world state is created (no silent empty board). Restore the file afterward.

Why

  • REQ-CARD-DATA-2 all run JSON loaded via Phaser's loader in preload, before any mint.
  • REQ-CARD-DATA-3 a failed load surfaces an error before game state is created.

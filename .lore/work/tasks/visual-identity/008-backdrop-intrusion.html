Backdrop & intensity-driven intrusion

REQ-VIS-8910

What

The reality base that slides toward intrusion as the run escalates.

  • New src/game/backdrop.ts — a persistent BackdropLayer: the reality image at the lowest depth (setDepth(-10), behind cards/HUD/overlays), plus the intrusion overlay above it. Neither is setInteractive, so pointer hits still reach cards and buttons.
  • Read intensity through the public seam — this.game_.intensity() — never import the core intensity module directly (that breaches the renderer's read-via-GameCore discipline).
  • Each drawAll: compute target alpha = intrusionForIntensity(this.game_.intensity()) and retarget the overlay's tween.
  • Compose the overlay (edge/radial weighting, or a masked-clear board zone) so a mundane boundary stays visible even at max intensity — the apocalypse stays localized.
Init: create the layer in create() with the same rule as S7 — after createGame(42) (line 83), before the win/loss bringToTop, never in the cardObjects destroy loop.
Tween (riskiest detail): retarget fire-and-forget on a single long-lived tween — Phaser 3.90 tween.updateTo('alpha', target, true) (or stop-and-restart). Do not call this.tweens.add(...) per repaint; that queues a second tween and lets the backdrop lag the authoritative state, which REQ-VIS-2/9 forbid.

Validation

  • Mundane at a fresh Act 1 / full HP; visibly more intruded (green) after advancing acts and losing HP.
  • The transition tweens, it does not snap; rapid dispatches retarget the same tween rather than stacking.
  • A mundane/intrusion boundary stays visible at maximum intensity (board area not consumed edge-to-edge).
  • Pointer input still works: clicks on cards, End Turn, Cancel/Confirm all register with the backdrop present.
  • The mapper (S4) is the only source of the alpha — no intensity math inlined in backdrop.ts.

Why

The backdrop expresses the world's reality (REQ-VIS-8); intrusion tracking the pure intensity read-model is the core of the look (REQ-VIS-9); the localized-apocalypse boundary is the signature contrast the visual direction names (REQ-VIS-10). See plan Step 8.

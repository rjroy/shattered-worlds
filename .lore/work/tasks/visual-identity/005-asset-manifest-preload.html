Asset manifest & Phaser preload

REQ-VIS-17

What

The one place that crosses from string keys to real URLs, so textures are loaded before create() draws anything.

  • src/game/assetManifest.ts: map each asset key → a Vite-imported URL (import realityUrl from './assets/themes/zombie-big-box/bigbox-reality.webp', etc.), exported as a Record<string, string>. Vite rewrites these with the /shattered-worlds/ base automatically, so nothing 404s on deploy.
  • Add TableScene.preload(): for the active theme's keys (selectTheme(state.worldId) → backdrop.realityKey, backdrop.intrusionKey, walker.textureKey) plus the shared cardback, call this.load.image(key, manifest[key]).
  • assetManifest.ts is Phaser/Vite-only and must never be imported by any test — its .webp imports do not resolve under bun test. Asset resolution is verified from build output, not by importing the manifest.

Validation

  • bun run dev loads with no missing-texture warnings in the console.
  • bun run build then bun run preview (serves dist/ with the /shattered-worlds/ base): every theme asset and the card back resolves — no 404 on the real subpath, which dev alone does not exercise.
  • No test file imports assetManifest.ts (grep the test suite).
  • bun test still green (the headless pure tests did not start importing the manifest).

Why

Assets must bundle and resolve on the Pages subpath with the pipeline unchanged (REQ-VIS-17), and the deployment-path lesson says hardcoded / paths 404 on a subpath — Vite imports avoid that. Isolating the manifest keeps the pure tests (S3/S4) headless. See plan Step 5.

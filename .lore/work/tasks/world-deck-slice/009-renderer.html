Phaser renderer & targeting grammar

REQ-WDS-2725

What

Make the world playable: a Phaser scene that renders the mixed hand and Hazards, plus a renderer-only selection state machine that turns clicks into whole core actions, driven entirely by availableActions. Full interaction grammar, no polish (skip shimmer/particles/intensity-FX/drag/keyboard).

  • selection.ts — the transient selection state machine (IDLE → SELECTED → resolve TargetSpec → COMMIT, with Cancel/Esc from any non-idle state). Holds only renderer state; never enters GameState. Emits a finished Action on commit.
  • render.ts — layout + factories: a player card vs a world card (visually distinct); a Hazard panel showing cost, current turn-Progress, keywords, penalty, reward; HP; current act + pile counts; the End Turn button; the world-discard zone; a modal chooser; a multi-select counter + Confirm; Cancel; win/loss screens. Target previews are plain-text badges (Progress incl. keyword bonus).
  • TableScene.ts — owns one GameCore; on each repaint reads state + availableActions(state) and paints affordances (playable, unplayable-dim, discardable, the Door's locked marker). Routes selection-machine commits to dispatch, then re-reads and repaints. End Turn is disabled while a selection is active; destructive actions (discard, Regroup destroy) are always two-step.
  • main.ts — likely unchanged; adjust scene wiring only if needed.

The renderer is the sole importer of phaser. Highlights are always a function of current state, re-read after every dispatch (never cached across one). The intensity signal exists but is not consumed for FX in this slice. A sub-agent may build the render.ts factories while the scene/selection wiring is built, then integrate.

Validation

  • typecheck + lint pass; the boundary lint confirms no phaser/DOM import leaked into src/core/.
  • bun run dev is playable: the browser shows the mixed hand with Hazards (cost, keywords, penalty, reward), HP, the act, pile counts, and End Turn. Playing cards at Hazards (with target/modal/multi-select where required), discarding a Hazard via the discard zone, and ending turns advances the loop to a win or loss screen.
  • Screenshot (via the verify/run flow) shows at least: the hand with one player card and one world card (visually distinct), the HP display, a dim unplayable card, and the End Turn button.
  • Selecting a card with no legal target is impossible (renders dim/unselectable); Cancel from any selection dispatches nothing.
⚠ Review gate (largest single surface): a fresh-context sub-agent reviews the renderer for boundary leaks, cached-affordance bugs, and any targeting logic that duplicates the core selector instead of reading it.

Why

REQ-WDS-27 (a Phaser renderer makes the world playable; clear correct interaction, not polish; the shared availableActions surface) and REQ-WDS-25 (events carry ids for animation). Realizes the targeting-interaction grammar. See plan Step 9.

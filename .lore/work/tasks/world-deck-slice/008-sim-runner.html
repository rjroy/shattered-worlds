Headless sim — the second driver

REQ-WDS-292

What

Prove "one core, two drivers": a random-legal policy built from availableActions drives many full worlds to a terminal state at full speed, preserving id-accounting and yielding the first balance data.

  • policy.ts — read availableActions(state), choose uniformly among: play a legal card (with a legal target/choice/returnIds drawn from legalTargets), discard a discardable Hazard, or end the turn. Choice randomness uses the seeded core RNG helpers, so a sim run is itself reproducible.
  • run.ts — run N worlds to won/lost (with a per-world action cap as a safety net), assert the id-accounting invariant every step, record metrics (turns, end HP, win/loss, cards acquired); exit non-zero on any throw or violation. Runnable via bun run sim.

The runner consumes only state, ignoring the event log, demonstrating the renderer is optional to the core. Because the policy only picks actions the selector returned, it never produces an illegal action; a throw is a real bug, not swallowed.

Validation

  • Many random-legal worlds complete without throwing, each ending won or lost, id-accounting preserved throughout, with metrics recorded.
  • At least one won outcome appears across the worlds (the forced-Door path is reachable).
  • bun run sim exits 0 and prints a short metrics summary.

Why

REQ-WDS-29 (the headless sim drives the new loop to terminal using only the core and a random-legal policy over the new action set, records metrics) and REQ-WDS-2 (id-accounting). Proves the one-core-two-drivers property and yields first balance data. See plan Step 8.

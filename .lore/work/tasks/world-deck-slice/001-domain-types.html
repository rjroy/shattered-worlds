Domain types & vocabularies

REQ-WDS-2345625

What

Replace the POC's trivial schemas with the world-deck domain model. Define, with no behaviour yet, the immutable shapes everything later is checked against:

  • The discriminated Card = PlayerCard | WorldCard union (a kind field). PlayerCard carries {id, name, effect}; WorldCard carries {id, name, cost, keywords, discardable, penalty, reward} (REQ-WDS-3, 4).
  • Effect as a closed discriminated union: DealProgress (base + optional {tag, amount} keyword bonus), Draw (player/world), Heal, ReturnWorldCards{min,max}, DestroyCardInHand{0,1}, DiscardThenDraw, AddCard{template,dest} (the spec's AddCard name — not GainCard), AddWorldCardToTop{template}, Modal{branches}, and exactly one Sequence{steps} wrapper for Barricade and Panic.
  • Penalty union: Damage, SkipDrawNextTurn, GainCard{template}, AddWorldCardToTop{template}, None. No CannotDiscard variant — the Door's no-discard rule is the discardable:false flag alone (REQ-WDS-4).
  • Reward union: GainCard, AddPlayerCardToTop, AddWorldCardToTop, SurviveWorld, None.
  • Keyword = 'Hidden' | 'Creature' | 'Slow'. CardTemplateId as a literal-union of catalog names so effects reference templates type-safely.
  • GameState: playerDraw, hand (mixed, holds both card kinds), playerDiscard, worldDraw (current act, ordered), acts (remaining-acts queue) + actIndex, progress: Record<CardId,number> (per-Hazard turn-Progress), hp, skipDrawNext, status: 'playing'|'won'|'lost', rng, and a monotonic nextId counter (the deterministic id source).
  • The renderer/sim read-model: TargetSpec (none, hazard{tag?}, modal, returnWorld, destroyHand, discardPlayer, compound) and the AvailableActions interface (playable, discardable, canEndTurn, legalTargets).
  • Action: PlayCard{cardId, targetId?, choice?, returnIds?, destroyId?, discardId?}, DiscardHazard{cardId}, EndTurn.
  • The full GameEvent union from REQ-WDS-25 (CardPlayed, ProgressDealt{hazardId,amount,hazardTurnTotal}, HazardResolved, HazardDiscarded, DamageDealt, DrawSkipped, CardGained{id,dest}, CardDestroyed, WorldCardsReturned{ids}, HpChanged{hp}, CardsDiscarded, DeckShuffled, ActAdvanced{act}, CardsDrawn{ids}, TurnEnded, WorldWon, WorldLost).
  • Keep src/core/errors.ts (IllegalActionError) unchanged.

Validation

  • bun run typecheck and bun run lint pass on the new types.
  • Boundary still live: temporarily adding import 'phaser' to a src/core/ file makes bun run lint fail; removing it passes again.
  • Every array field is readonly; no any, no non-null assertions (consult the typescript-quality skill).
  • The Effect union uses AddCard (not GainCard) for the player effect; the Penalty union has no CannotDiscard variant.

Why

These schemas are what the rules, selector, reducer, renderer, and sim are all checked against. REQ-WDS-2 (immutable, uniquely-identified cards), REQ-WDS-3 (effect-bearing player cards), REQ-WDS-4 (Hazard cost/keywords/penalty/reward), REQ-WDS-5 (keyword tags), REQ-WDS-6 (zones/HP/progress/acts), and REQ-WDS-25 (animation-sufficient event log). The AddCard name and the dropped CannotDiscard variant resolve review findings before the types are frozen. See plan Step 1.

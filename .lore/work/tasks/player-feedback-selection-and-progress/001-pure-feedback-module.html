Pure feedback module: ring fraction + connector-style resolver

pending REQ-FEEDBACK-7 REQ-FEEDBACK-3 REQ-FEEDBACK-10

Dependencies: none. Parallel-safe with S2. Foundation for S4 (ring) and S8 (connector style).

What

Create a new Phaser-free module so the load-bearing math is unit-testable independent of the renderer, satisfying the spec's "pure function" demands and the structural boundary (no Phaser, core types only).

  • ringFraction(progress: number, cost: number): number → cost > 0 ? Math.min(1, Math.max(0, Math.min(progress, cost) / cost)) : 0. Guards divide-by-zero.
  • type ConnectorStyle = 'progress' | 'destroy' | 'return' and selectConnectorStyle(effect: CardEffect): ConnectorStyle | null — keyed off effect kind, looking through Modal/Sequence the same way dealProgressOf does (describe.ts:112): a reachable DealProgress → 'progress'; DestroyCardInHand → 'destroy'; ReturnWorldCards → 'return'.
  • Keep previewPlay (describe.ts:91) as the single place that turns (card, target, progress) into a sentence; ringFraction only formats the same numbers (FEEDBACK-10). The agreement test enforces this.

New module (not describe.ts) for cohesion: describe.ts is the English-string layer; this is ring/connector math. Both stay headless-testable.

Validation

  • ringFraction unit tests: returns 0 at progress 0; 0.5 at 1/2; 1.0 at progress≥cost; clamps (progress 3, cost 2 → 1.0); correct at the cost-10 boss (progress 5, cost 10 → 0.5).
  • selectConnectorStyle unit tests: DealProgress → 'progress'; DestroyCardInHand → 'destroy'; ReturnWorldCards → 'return'; resolves through a Modal branch and a Sequence step.
  • Agreement test (FEEDBACK-10): sample at minimum progress = cost−1 ("not yet"), progress = cost ("clears"), progress = cost+1 ("already over"); ringFraction reaches 1.0 exactly when previewPlay reports "clears".
  • bun test green; tsc/eslint clean; module imports core types only (no Phaser, no core values).

Why

Satisfies the pure-function requirements of REQ-FEEDBACK-7 (ring fraction), REQ-FEEDBACK-3 (data→style mapping), and REQ-FEEDBACK-10 (single source of truth). Per plan step S1.

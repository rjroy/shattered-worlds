Decorate the connector by targeting type

pending REQ-FEEDBACK-3

Dependencies: S7 (connector geometry), S1 (selectConnectorStyle), S2 (connector palette).

What

Style the connector from selectConnectorStyle (S1) using the theme palette (S2): a small, learnable three-style vocabulary carrying what the action does, not just where it lands.

  • progress → accent straight line feeding into the target's cost ring (connectorProgress, paired with ringAccent).
  • destroy → harsh/jagged red line (connectorDestroy).
  • back to deck (return) → curved arrow looping toward the world-deck pile (connectorReturn). Get the deck endpoint via a small accessor on PileLayer (e.g. pileLayer.worldPileCenter(): {x,y}) rather than hardcoding coordinates.
  • Resolve the style from the acting card's effect for the current step via selectConnectorStyle. The data→style mapping is authoritatively gated by S1's unit test; this task only renders the three.

All three phases are reachable in a normal Zombie Big-Box run: DealProgress (Explore) on a hazard; ReturnWorldCards (Barricade/Adrenaline) on world cards; DestroyCardInHand (Zombie Big-Box destroy) on a hand card.

Validation

  • Mapping is gated by S1's selectConnectorStyle unit test (already green).
  • Manual: trigger a DealProgress card on a hazard → accent line into the ring.
  • Manual: trigger ReturnWorldCards (Barricade/Adrenaline) on a world card → looping arrow toward the deck.
  • Manual: trigger the Zombie Big-Box DestroyCardInHand effect on a hand card → jagged red line.
  • bun test green; tsc/eslint clean.

Why

Implements REQ-FEEDBACK-3 — the connector pre-reads the effect (chips / kills / bounces), making the link do double duty: disambiguate the target and preview intent. Per plan step S8.

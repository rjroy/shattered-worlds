S6

Split instruction hint and live preview into two surfaces

pending REQ-FEEDBACK-5

Dependencies: S3 (placed after, since it edits TableScene.ts; logically independent of S4/S5).

What

Today one bottom-center selectionHint carries both the phase instruction and the hover preview, so they clobber each other. Give the preview its own surface.

  • Add a second persistent Text (e.g. previewSlot). updateHint writes only the phase instruction to selectionHint; showTargetPreview writes only to previewSlot. The two must never collapse into one slot.
  • pointerout contract: pointerout still calls updateHint() (instruction text stays stable in its own slot) and additionally clears the preview via previewSlot.setText(''). The old behavior where pointerout re-ran updateHint to overwrite the preview is gone.
  • Preview string stays sourced from previewPlay (FEEDBACK-10; honors self-describing-card-faces).
  • Placement: default to a dedicated fixed preview slot (simplest; degrades on touch where there is no hover). If a pointer-following floating slot is used instead, it must fall back to a stable position when there is no hover and the target's own tap-to-preview must still work — but the two surfaces never merge.

Validation

  • Manual: hover a target during a targeting phase → the instruction text and the live preview are both visible at once; neither overwrites the other.
  • Manual: hover-out clears the preview while the instruction text remains.
  • Manual (touch sanity): with no hover, the two surfaces do not collapse into one shared slot.
  • bun test green; tsc/eslint clean.

Why

Implements REQ-FEEDBACK-5 — instruction and live preview must occupy distinct surfaces so one does not overwrite the other. Per plan step S6.

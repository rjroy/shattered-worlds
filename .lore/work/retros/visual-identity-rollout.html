Surprises

Surprise

The concurrent TableScene.ts edits worked. Three parallel agents, each writing different sections of the same 693-line file via exact-string Edit calls — and all three sets of changes were present in the final file with no manual merge. This was the highest-risk coordination point in the session. It worked because the agents were given edit targets in different methods. The precondition for this going wrong is two agents targeting the same method with overlapping match strings; when that happens, the last writer silently wins.

Surprise

The intrusion overlay has no alpha channel. The plan said the overlay should have a "transparent center" so the board area stays readable. The art-gen output produced a webp (VP8 lossy, no alpha) where the center is compositionally bright/clear and the edges are the fluorescent green intrusion. This achieves the visual intent through image content rather than pixel transparency. The spec task language was "compose the overlay (edge/radial weighting, or a masked-clear board zone)" — both are valid, and the image-content approach was generated rather than specified. It works, but any future generator targeting "transparent center with alpha" will get a different kind of file. Worth noting for the next world's asset brief.

Surprise

preload() cannot read state.worldId. The plan intended preload() to call selectTheme(state.worldId) to load the right assets dynamically. But Phaser calls preload() before create(), so this.game_ doesn't exist yet. The implementation hardcodes selectTheme('zombie-big-box') in preload() and reads from state in create(). This is architecturally unavoidable for the single-world demo but becomes a real constraint if multiple worlds ever need different asset sets — the worldId would need to be passed as Phaser scene init data so it's available in preload().

Surprise

Phaser 3.90 types updateTo correctly — no cast needed. The plan noted "if TypeScript cannot resolve updateTo on the Phaser.Tweens.Tween type, cast as any." In practice, Phaser 3.90's bundled types fully declare updateTo(key: string, value: number, startToCurrent?: boolean): this. No cast was needed.

Dead ends

Dead end

The intrusion tween was initially built wrong. The original constructor created a tween targeting alpha: 0 → 0 (start and end identical) with a 1500ms duration. This tween completes after 1.5 seconds and Phaser recycles the Tween object back into its pool. The update() method then called updateTo on the stale reference every drawAll(). In Phaser 3.90's tween pool, a recycled Tween can be assigned to a new target object; calling updateTo('alpha', ...) on it would change that other object's alpha tween target. The pattern was changed post-commit to killTweensOf + scene.tweens.add, which avoids holding a stale reference entirely. This change was triggered by a user report of clicking issues, but the exact causal connection was not established before commit.

Open: clicking issue (unresolved)

Open

After the implementation was committed and the dev server running, the user reported trouble clicking on things. The investigation in-session was extensive but inconclusive: non-interactive Phaser objects (backdrop images, pile container images) cannot block input in the Phaser 3 input model; depth sorting is enabled and depth-0 objects receive input before depth-−10 objects; card containers are the last objects added to the display list (highest index within depth 0) so they are checked first. No static-analysis cause was found. The tween recycling issue was patched as a precautionary measure. Browser console errors and hand-cursor-on-hover behavior were not confirmed because browser navigation was blocked during the session. The issue is flagged in PR #2 and remains open.

What drifted from the plan

  • preload() hardcodes worldId — architecturally unavoidable; plan assumed dynamic read from state.
  • walkerPresentation called with hasWalker: true always in backdrop.ts rather than derived from theme.walker !== undefined. Functionally harmless: the sprite-undefined guard fires first. The hidden branch of walkerPresentation is dead code in the current wiring.
  • GrainSpec and intrusionHue defined but not rendered — intentional deferral of REQ-VIS-14 (Should), noted in the plan as out of scope.
  • Walker tween uses stop-and-recreate, not updateTo — both approaches were listed as valid in the plan; the intrusion tween issue pushed toward the same pattern for consistency.

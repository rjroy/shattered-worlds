Phase log

Phase 1: selection.ts
Implemented pure state machine. States: idle, selected, awaiting-hazard, awaiting-return, awaiting-discard, awaiting-destroy, awaiting-modal. All transitions are pure functions returning new state. buildAction assembles the core PlayCard Action when complete. IDLE constant exported. No GameState dependency.
Decision: awaiting-hazard carries an optional targetId so that compound steps (Barricade) can store the hazard target before transitioning to awaiting-return. This avoids a separate intermediate state variant.
Phase 2: render.ts
Factory functions create Phaser Containers for player cards (dark blue, 120×160) and world/hazard cards (dark red, cost centred large, keywords, penalty/reward). HUD creates four text objects; updateHUD writes current values. Win/loss screens are full-screen overlays hidden by default. End Turn, Cancel, Confirm buttons created as Text objects with pointer cursors.
Decision: applyCardHighlight and dimCard operate on the bg rectangle (index 0 in container children list) rather than re-creating objects, so highlights are cheap to apply on every repaint.
Phase 3: TableScene.ts
Scene owns one GameCore (seed 42). drawAll() destroys all card containers, recreates them, computes highlights from availableActions fresh, and updates HUD. No affordance state cached across dispatches.
Decision: world cards and player cards split into two rows (WORLD_ROW_Y=200, HAND_ROW_Y=430), each centred dynamically.
Decision: clicking a discardable world card while idle routes to onDiscardClick immediately (single-click discard in idle state). Clicking a player card while idle routes to the selection/targeting path. This satisfies the two-step discard requirement: only discardable cards (flagged by availableActions) respond; non-discardable world cards are simply not interactive for discard.
Decision: compound spec (Barricade) handled inline: after hazard target click, scene transitions sel to awaiting-return with the hazard's targetId saved, then waits for Confirm.
Decision: Sprint modal chooser built inline in showModalChooser. Branch 1 (Slow bonus) checks legalTargets at step 1; if empty, button renders dimmed. Sprint's choice=0 branch (Draw) goes to selected phase which immediately dispatches with choice: 0.
Phase 4: typecheck + lint
Fix 1: Redundant sel.phase !== 'idle' guard inside a 'cardId' in sel branch — TypeScript correctly flags that after the in check the phase can never be 'idle'. Removed the redundant guard.
Fix 2: ROW_LEFT constant defined but unused (layout is centred dynamically). Commented out with a note.
Fix 3: branchLabel parameters available and cardId unused (labels are Sprint-specific hardcoded strings plus a generic fallback). Renamed to _available and _cardId per the project convention.
After fixes: bun run typecheck clean, bun run lint clean, 114/114 tests pass (all in core — renderer has no unit tests, consistent with the Phaser/DOM separation).

Simplifications vs spec

  • Modal branch labels are Sprint-specific strings (idx 0 / 1) with a generic fallback, rather than synthesised from the Effect tree. Sufficient for the single modal card in this slice; extend when more modal cards exist.
  • Discard zone is implemented as a single-click on a discardable world card while idle, not a separate spatial zone. Interaction grammar is identical (one deliberate click after seeing the card is discardable); the spatial separation was polish.
  • No progress bar per hazard card in the renderer — progress values are in state.progress keyed by card id but were not wired to a visual element. Adding a progress badge to createCardObject is straightforward.

◈ Decisions this plan makes (review these first)

The brainstorm left small concrete choices open. This plan fixes them so implementation never guesses. Each is cheap to change now and expensive to change mid-phase.

D1canExile defaults to true and is read on every world card, uniformly. Optional field on WorldCardTemplate (canExile?: boolean), minted onto WorldCard as template.canExile ?? true — the same pattern as exhaust ?? false but inverted polarity, because exilability is the norm. Door and The Walker get explicit "canExile": false in starter.json. Per the 2026-06-10 discussion: Door/Walker living in shared starter state is intentional, and the flag is a general capability (future meta-progression may enable exile in shards where it isn't expected today), so no world-specific special-casing anywhere in the engine.
D2Exile operates on the current worldDraw pile only. It never advances acts (you can't outrun road you haven't reached) and stops gracefully when fewer exilable cards exist than amount. Non-exilable cards are skipped in place, order preserved.
D3Brace charges persist across turns until consumed. braceCharges is a plain counter (like energy carryover, REQ-ENERGY-6 precedent). resolveForceDestroy consumes min(braceCharges, pendingForceDestroy) charges first; only the unabsorbed remainder destroys cards. Note: the meta-progression brainstorm uses "Brace" as a bird-building card name — a card named Brace granting Brace charges is coherent, not a conflict; flagged so nobody "fixes" it.
D4Playability rules for the new effects: DealProgressAll is playable only when ≥1 world card is in hand; ExileTopWorldCards only when worldDraw contains ≥1 exilable card; Brace always (charges bank). No dead plays, and the rule is statable in a sentence (Principle 5).
D5New-card sources (which world card grants what): zombie — new fetch Find Shotgun (act 3, Hidden) grants Shotgun; new Echoing Aisles (acts 2–3, Hidden) grants Shelf Sweep. Volcano — Gridlock's onCleared grants Nitro (clearing the jam gets you moving). Bird — Sliding Debris's onCleared grants Cut It Loose (cleared debris is something to throw). All numbers in the card tables below are starting values, tunable per Principle 6.
D6New events: BraceChanged {braceCharges} (on gain, mirrors EnergyChanged), BraceConsumed {absorbed, remaining}, WorldCardsExiled {ids}. The renderer is state-diff based and ignores the event stream (ForceDestroy retro), so these are for tests and future juice — adding them is safe and costs nothing.

◈ Phase 3 — DealProgressAll core change

P3

depends on: P2 (sequencing only — phases are independent in code but land smallest-risk first)
Core changes

◈ Phase 4 — ExileTopWorldCards + canExile core change

P4

The bypass: volcano's signature verb, and the one with livelock teeth

depends on: P3 (sequencing); risk-wise the last of wave 1 by design
Core changes
  • src/core/model/cards.ts: canExile?: boolean on WorldCardTemplate; canExile: boolean on WorldCard (types.ts); mintCard stamps template.canExile ?? true (D1).
  • src/data/worlds/starter.json: "canExile": false on Door and The Walker. No other template gets the field (default covers them) — and no engine code may branch on world id or template name for exile (D1: general capability).
  • types.ts: { kind: 'ExileTopWorldCards'; amount: number } + WorldCardsExiled {ids} event.
  • effects.ts: walk worldDraw from index 0; collect exilable cards until amount spent; rebuild the pile with non-exilable cards in their original relative order; never touch acts (D2). Exiled cards go to no zone. Emit one WorldCardsExiled with the removed ids.
  • available.ts: spec none; isPlayable → worldDraw.some(c => c.canExile) (D4).
  • describe.ts: "Exile the top 2 cards of the world deck"; no preview math needed.
  • sim/policy.ts: spec-none confirmation.
Data: highway-volcano.json Floor It rework
  • Floor It: "exhaust": true, energyCost 0, effect → { "kind": "ExileTopWorldCards", "amount": 2 }. Still granted by Find a Vehicle. The one-shot nitro escape.
Livelock-guard tests — the explicit risk register for this phase
  • Exile empties worldDraw while acts remain → next refill advances the act normally; no false WorldLost.
  • Exile when worldDraw is only non-exilable cards → unplayable per D4 (assert in available.test).
  • Exile leaves only The Walker in worldDraw, acts empty → game continues; Walker still reachable; guard B does not fire (a world card exists).
  • Exile with amount > exilable count → removes what it can, stops gracefully, deck order of survivors intact.
  • Determinism: exile is order-based, no RNG — assert byte-identical state for same seed+actions (golden-style unit, not just the golden suite).
Validation gate bun test (incl. all five guard cases above) · tsc --noEmit · lint · goldens reviewed. Manual playtest of the volcano end-to-end including a Floor It win line. Fresh-context code review of phases 2–4 together (the whole wave-1 engine surface).

◈ Cross-cutting rules (apply to every phase)

RuleWhy / source
The add-an-effect checklist is: types.ts union → effects.ts case → available.ts (3 functions) → describe.ts → sim/policy.ts check → tests in effects/available/reduce suites.ForceDestroy retro; describe has no default arm and fails compile until handled — that trip-wire is a feature, don't add a default.
isPlayable has default: return false — a missing case is a silent bug, not a compile error. The card simply never appears playable. Every new effect kind's test suite must assert the card appears in availableActions(state).playable when its conditions are met, not only that applyEffect behaves.Plan review finding; unlike describe/applyEffect, this switch fails silently.
No Phaser imports anywhere in P2–P4/P6 work; everything lands in src/core/ + JSON. Renderer is untouched (state-diff repaint handles all of it).Lint-enforced core/game boundary (CLAUDE.md); ForceDestroy retro confirmed events are ignored by the renderer.
Determinism: no new RNG anywhere in these effects. Exile is order-based; sweep is iteration-based; brace is arithmetic.Core seedability invariant (CLAUDE.md).
Golden tests are regenerated only with a reviewed diff, never blindly, and only in phases that change data or turn flow.golden.test.ts exists to catch exactly this class of change.
